Imagine perder todo o conteúdo do seu site de uma hora para outra — textos, imagens, configurações, tudo. Parece pesadelo? Sem **backup**, isso pode acontecer. Um ataque, erro humano ou falha no servidor pode apagar seu site inteiro.

Neste artigo, vamos explicar **o que é backup** e como garantir que seu site esteja sempre protegido.

## O que é backup?

**Backup** é uma **cópia de segurança** dos arquivos e dados do seu site, armazenada em local separado. Se algo der errado, você pode restaurar o site a partir do backup, voltando ao estado anterior.

É como fazer uma cópia das chaves de casa — se perder a original, tem a reserva.

## Por que fazer backup?

- **Ataque hacker**: site infectado pode ser restaurado
- **Erro humano**: deletou algo por engano? Restaure
- **Falha do servidor**: problemas técnicos podem corromper dados
- **Atualização que deu errado**: volte à versão anterior
- **Migração**: facilita mudar de hospedagem

## Com que frequência fazer backup?

| Tipo de site | Frequência recomendada |
|-------------|----------------------|
| Blog com publicações frequentes | Diário |
| Site institucional | Semanal |
| Loja virtual | Diário |
| Site pouco atualizado | Quinzenal |

## Como fazer backup

### Sites em plataformas (Linkfl, Wix)
O backup geralmente é **automático** e gerenciado pela plataforma.

### WordPress
- Plugins: UpdraftPlus, BackWPup, Duplicator
- Via hospedagem: muitas oferecem backup automático no painel
- Manual: baixar arquivos + exportar banco de dados

### Onde armazenar
- Google Drive
- Dropbox
- Amazon S3
- No computador local (como backup adicional)

**Regra de ouro**: nunca armazene o backup no mesmo servidor do site. Se o servidor falhar, você perde ambos.

## Perguntas frequentes (FAQ)

### Backup automático é melhor que manual?

Sim, pois garante consistência. Backups manuais dependem da sua memória e disciplina.

### O Linkfl faz backup automático?

Sim. Plataformas gerenciadas como o Linkfl cuidam dos backups automaticamente.

### Backup ocupa muito espaço?

Depende do tamanho do site. Um site institucional típico ocupa 50-500 MB. Serviços de nuvem oferecem espaço suficiente para vários backups.
